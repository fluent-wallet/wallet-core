## WalletCore 是什么？ {#what-is-WalletCore}

WalletCore 是一个钱包核心功能库，专为构建快速 web3钱包 而设计。内置了钱包的 账户管理/资产管理/交易签发/交易历史 等核心功能，实现了部分热门链的支持并允许自定义扩展接入其他未内置实现的链。

#### WalletCore 支持跨端吗？

WalletCore的存储层基于 [RxDB](https://rxdb.info/)，RXDB不是一个自包含的数据库，各端都有支持的存储层的接入(可能需要为一些高性能的选择付费，不过 移动端/插件端/web端 都有免费的存储层可选），详情可见 [Rxstoarge Layer](https://rxdb.info/rx-storage.html)。

#### WalletCore 如何保证助记词/私钥的安全？

WalletCore 提供一个 encryptor 的机制，允许自定义对助记词的加解密方式，也提供了内置的两种加解密方式：

- 签发时密码输入(不存储密码，每次签发交易时都需要输入密码)
- 内存密码常驻(解锁钱包时将密码存储在内存中，锁定后钱包后清空)

#### WalletCore 如何支持列表中没有的链？

WalletCore 内置了部分热门链的支持，未内置的新链只需要实现 chains 中的 Base 抽象类即可。

详细见 [支持一条新链](../extend)。


#### 我需要自己实现钱包的UI吗？

是的，WalletCore 只提供钱包的核心功能，UI 需要自己实现(可以参考 Example 的大致流程)。并且需要选择/组合/实现一些钱包的细节feature，比如: 
- 插件钱包 常见的 密码常驻模式，或者 mobile钱包 常见的 签发时密码输入模式。
- rainbow钱包 式的初始化即备份助记词，或者 将助记词备份流程放在初始化钱包完成后的首页提示中(与非初始化创建新助记词账户一致)。
- 初始化时候，是先初始化密码还是先创建助记词账户。如果没走完全部流程直接关闭钱包，下次打开时候是否需要重新走一遍流程，等等。


#### Wallet-Connect / 硬件钱包接入 / window注入(ERC6936) 等外接功能 WalletCore 支持吗？

是的，WalletCore 提供了这些常见功能的实现，搭起一个覆盖常见功能的 web3钱包 只需要组装配置即可。
